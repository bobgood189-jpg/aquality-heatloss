# Безопасность Aquality | WaterPro

Калькулятор — **статическое клиентское приложение** (один `index.html`, данные в
`localStorage`, без своего сервера). Ниже — что уже сделано, что принципиально
нельзя защитить на чистом клиенте и что нужно настроить руками на хостинге.

---

## ✅ Что защищено в коде

| Слой | Было | Стало |
|------|------|-------|
| Пароль владельца | `atob('UC5t…')` → `P.maxmud74` открытым текстом | Только необратимая запись **PBKDF2-SHA256 + соль** (`OWNER_CRED`). По коду пароль не восстановить. |
| Пароли пользователей | base64 «хэш» (обратим) / иногда plaintext | **PBKDF2-SHA256 + соль на пользователя** (`AQSec.hashPassword`). Старые записи авто-мигрируют при первом входе. |
| Перебор пароля | не ограничен | Лимит попыток + растущая задержка (после 5 неудач: 2с→…, потолок 5 мин), состояние в `aq_login_throttle_v1`. |
| Сессия | `aq_session_v3 = "<id>"` (вечно) | Токен со сроком: `{id, t, exp}`, авто-истечение 7 дней. |
| Заявки (PII) | plaintext JSON в `aq_leads_v1` | **AES-256-GCM** на диске (`AQVault`), чтение прозрачное. Ключ — per-device (`aq_dk_v1`). |
| Заголовки | нет | `<meta>` CSP + `referrer` в `index.html`; полный набор в `_headers`. |

Криптография — нативный **Web Crypto API**, без внешних библиотек. Работает офлайн
(`file://`) в современных браузерах; при отсутствии `crypto.subtle` мягкая деградация.

### Сменить пароль владельца
```bash
node scripts/encrypt.js owner-hash 'НовыйПароль'
# вставьте выведенный  const OWNER_CRED = {...};  в index.html
```

## ⚠️ Что НЕЛЬЗЯ защитить на чистом клиенте (честно)

- **Права владельца/админа.** Любая проверка прав и любой ключ, которыми браузер
  пользуется автоматически, **извлекаемы** из исходника. PBKDF2 замедляет подбор,
  срок сессии убирает тривиальный `=true`, но человек с DevTools всё равно может
  локально подделать сессию. **Настоящее решение — перенос проверки прав на бэкенд**
  (в проекте уже подготовлен Supabase Auth + RLS: заполните `SUPABASE_URL`/`ANON_KEY`).
- **Шифрование заявок — «обфускация-grade».** Заявки пишет анонимный посетитель
  (пароля нет), поэтому ключ выводится из per-device секрета и лежит на устройстве.
  Это убирает plaintext из `localStorage`/бэкапа-экспорта, но не спасёт от того, кто
  открыл исходник. Сильное (password-derived) шифрование возможно только для данных
  залогиненного пользователя — это следующий шаг, см. ниже.
- **Скрыть код полностью нельзя.** Минификация/обфускация лишь усложняют чтение и
  копирование; JS всё равно исполняется в браузере и обратим.

### Не зашифровано сейчас (осознанный остаток)
`aq_users_v3` (телефоны/почты — но **пароли там уже PBKDF2**), `aq_addresses_*`,
аватары. Не трогали ради стабильности (десятки синхронных мест чтения). Пароли —
главная ценность — защищены. Шифрование этих полей под паролем пользователя —
кандидат на следующую итерацию.

## 🌐 HTTPS-публикация (вручную)

Сайт уже на **Netlify** (`aqualityheatloss.netlify.app`) → HTTPS включён автоматически.
Файл `_headers` (в корне публикуемой папки) задаёт жёсткие заголовки.

### Netlify
1. Залейте корень репозитория (с `index.html` и `_headers`).
2. HTTPS работает из коробки. На дефолтном поддомене `*.netlify.app` отдаётся
   **общий wildcard-сертификат Netlify** (издатель — DigiCert, CN `*.netlify.app`).
   Персональный **Let's Encrypt**-сертификат Netlify выпускает автоматически только
   когда привязан **кастомный домен** (Domain settings → Add custom domain).
3. Domain settings → **Force HTTPS** (редирект http→https).
4. Проверьте заголовки: https://securityheaders.com — должно быть A/A+.

### Cloudflare Pages
- Тоже читает `_headers`. SSL/TLS → режим **Full (strict)**, включите **Always Use HTTPS**.

### GitHub Pages
- ⚠️ Кастомные заголовки **не поддерживаются** — работает только `<meta>`-CSP из `index.html`.
- Settings → Pages → **Enforce HTTPS**. Для полноценных заголовков поставьте Cloudflare перед Pages.

`Strict-Transport-Security` в `_headers` уже **включён** (`max-age=31536000;
includeSubDomains; preload`) и подтверждён в живом ответе сайта. Учтите: пока он
активен, откат на HTTP недоступен на год — это осознанный выбор «навсегда на HTTPS».

## 🔒 Опционально: «замок» на весь сайт
`scripts/encrypt.js` шифрует весь `index.html` (AES-256-GCM + PBKDF2) в
`index.protected.html` — страница просит пароль и хранит контент зашифрованным.
Подходит для закрытой версии, **не** для публичного калькулятора (один общий пароль
на вход). Использование: `SITE_PASSWORD='пароль' node scripts/encrypt.js`.
