# Доказательства проверки в браузере

Проверка выполнена в браузере через локальный сервер (Node static server, порт 8788)
и Claude Preview (Playwright/Chromium). Ниже — фактические результаты `eval` в
контексте страницы. Скриншоты сделаны в ходе сессии (login-форма в режиме Supabase
показывает «Email / you@example.com»; главная страница сохраняет премиум-вид).

| totalW | 6360 | **6360** | — |
| totalKw | 6.3601 | **6.3601** | **6.3601** |
| boilerKw | 7.9501 | **7.9501** | — |

## 2. Загрузка Supabase JS + конфиг
```json
{"sbLibLoaded":true,"SB_CONFIGURED":false,"aqUser":null,
 "hasAqSaveCalc":true,"hasAuthRouter":true,"hasLegacy":true}
```
✅ `@supabase/supabase-js@2` (CDN UMD) загружается; с плейсхолдерами `SB_CONFIGURED=false` (офлайн).

## 3. Офлайн-fallback: регистрация + текущий пользователь
```json

```
✅ Регистрация работает; `authCurrentUser()` возвращает пользователя; форма редактирования профиля рендерится.

## 5. Supabase-режим (dummy-конфиг, без сети): не падает, UI адаптируется
```json
{"SB_CONFIGURED":true,"clientInit":true,"sessionRestore":"null-ok",
 "loginUsesEmail":true,"registerEmailRequired":true,"totalKw":6.3601}
```
✅ Клиент инициализируется; восстановление сессии из localStorage не делает сети и
возвращает `null` без ошибок (это и есть механизм «сессия переживает перезагрузку»);
форма входа переключается на Email; в регистрации Email становится обязательным.

## 6. Устойчивость лид-захвата (Задача D)
```json
{"leadInsertResilience":"error-handled-gracefully","authOverlayOpen":true}
```
✅ При недоступном Supabase `insert` в `leads` **не бросает синхронно**, ошибка
логируется — значит WhatsApp всё равно откроется (`window.open` идёт следом синхронно).

## 7. Консоль чистая
`preview_console_logs level=error` → **No console logs** (и в офлайн-, и в Supabase-режиме).
Единственное предупреждение — стандартное «cdn.tailwindcss.com should not be used in
production» (было и раньше, не ошибка).

## Что НЕ проверено вживую (нужны ключи владельца)
Реальный сетевой round-trip к Supabase (фактическая запись строки в `profiles`/
`calculations`/`leads` и чтение её обратно) требует боевого проекта Supabase с
`SUPABASE_URL` + `anon key`, которых в этой среде нет. Код к этому готов и
проверен на уровне: инициализация клиента, построение запросов, обработка ошибок,
переключение режима, восстановление сессии. После вставки ключей владельцем и
выполнения `supabase/schema.sql` путь становится боевым.
**Чек-лист приёмки владельцем — в `PROGRESS.md`.**
