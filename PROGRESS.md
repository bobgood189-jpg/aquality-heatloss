Воспроизведение кликами (Playwright `elementFromPoint` + dispatch PointerEvent на канву,
логин владельцем `Hasanboy.admin`):

| Раздел | Симптом у пользователя | КОРНЕВАЯ ПРИЧИНА (найдена, не угадана) |
|---|---|---|
| Меню «Мой профиль» | клик не реагирует | **stacking-context trap**: `#user-dropdown` (`z-index:90`) лежит внутри `<nav class="relative z-10">` (стр. 425). Сосед `#hero-content` (`relative z-10`, стр. 440) идёт ПОЗЖЕ в DOM и при равном z-index перекрывает весь контекст nav. Пункты дропдауна свисают ниже полосы nav (y>73px) в зону hero-content → `elementFromPoint` возвращает `hero-content`, клик не доходит до пункта. |
| Меню «Мои расчёты» | клик не реагирует | то же (та же причина) |
| Меню «Панель администратора» | клик не реагирует | то же (та же причина) |
| Инструмент «Панорама» (hand) | клик не реагирует | кнопка в категории ленты **'view', свёрнутой по умолчанию** (стр. 3679: `open = … : (cat.id!=='view')`). `er-body` получает класс `hidden` → `display:none`. Кнопку физически нельзя кликнуть. **Сама логика pan корректна** (edDown стр.3945 ставит `EDITOR.pan`, edMove стр.4059 двигает `offX/offY`). |
| Инструмент «Приблизить» (zoom-in) | клик не реагирует | то же — кнопка в свёрнутой 'view'. Логика `edZoom` (стр.3774) корректна. |
| Инструмент «Расстояние» (distance) | «ничего не остаётся» | кнопка видна (категория 'measure' открыта) и считает длину, но по ТЗ должна **рисовать линию + подпись**, а текущая реализация (стр.3993) только показывает тост на 1.5с → воспринимается как «не работает». |
| Инструмент «Периметр» (perimeter) | реагирует не так | работает по выбранной комнате (тост «Периметр: X м»), но по ТЗ нужен режим **серия кликов → замкнутый контур → суммарный периметр**. |

Доказательства (Playwright):
- Меню: `elementFromPoint` в координатах всех 3 пунктов → `hero-content` (а не пункт).
- После принудительного раскрытия 'view' клики по всем 4 инструментам сработали:
  zoom-in 56.1→67.3; pan offset [-38,6]→[92,76]; distance тост «3.35 м»;
  perimeter тост «14.00 м» (комната 4×3 — верно). Консоль чиста.

   пункты дропдауна перехватывают клик корректно, оверлеи по-прежнему перекрывают nav. ✅ ПРИМЕНЕНО
2. **Лента редактора** (`buildRibbon`, стр. 5023): дефолт `(cat.id!=='view')` → `true` —
   все категории, включая «Вид», открыты по умолчанию → «Панорама»/«Приблизить» видимы. ✅ ПРИМЕНЕНО
3. **events формат** (`openProfile`, стр. 1760): `aq_events_v1` хранится как объект `{тип:count}`,
   но код ожидал массив → `events.filter is not a function`. Исправлено: `Array.isArray()` guard +
   `Object.entries().flatMap()` конвертация в массив `[{type}]` для совместимости. ✅ ПРИМЕНЕНО
4. **distance** — постоянная линия: не применено (в текущей сессии не требовалось).
5. **perimeter** — режим серии кликов: не применено (в текущей сессии не требовалось).
