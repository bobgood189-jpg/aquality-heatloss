# Подключение бэкенда Supabase — инструкция для владельца

Сайт — статика (один `index.html`). Роль бэкенда (аккаунты, профили, сохранённые
расчёты, заявки) выполняет **Supabase** (Auth + Postgres + RLS). Пока ключи не
вставлены, сайт работает в **офлайн-режиме**: калькулятор считает полностью, а
аккаунт и заявки сохраняются локально в браузере. Чтобы включить реальный бэкенд —
3 шага:

## Шаг 1. Создать проект Supabase
1. Зайдите на https://supabase.com → **New project** (бесплатного тарифа достаточно).
2. Дождитесь создания (1–2 мин).

## Шаг 2. Выполнить схему БД
1. В проекте: **SQL Editor → New query**.
2. Откройте файл [`supabase/schema.sql`](supabase/schema.sql), скопируйте всё, вставьте и нажмите **Run**.
   Это создаст таблицы `profiles`, `calculations`, `leads`, включит RLS и политики,
   и добавит триггер автосоздания профиля при регистрации.

## Шаг 3. Вставить ключи в сайт
1. В Supabase: **Settings → API**. Скопируйте:
   - **Project URL** (вид `https://xxxx.supabase.co`)
   - ключ **anon public** (НЕ `service_role`!)
2. Откройте `index.html`, в самом верху (в `<head>`) найдите блок:
   ```js
   window.AQ_CONFIG = {
     SUPABASE_URL: 'YOUR_SUPABASE_URL',
     SUPABASE_ANON_KEY: 'YOUR_SUPABASE_ANON_KEY'
   };
   ```
   Вставьте свои значения вместо плейсхолдеров. Сохраните, залейте на Netlify.

> ⚠️ В клиенте используется **только anon-ключ** — это нормально и безопасно: доступ
> ограничен политиками RLS. **`service_role`-ключ НИКОГДА не вставляйте в `index.html`.**

## Шаг 4 (один раз). Сделать себя владельцем
1. Зарегистрируйтесь на сайте обычным образом (email + пароль).
2. В Supabase **SQL Editor** выполните (подставьте свой email):
   ```sql
   update public.profiles set role = 'owner' where email = 'ВАШ_EMAIL';
   ```
   После этого ваш аккаунт получит доступ к админ-панели (Заявки, Пользователи).

## Подтверждение email (важно)
По умолчанию Supabase требует подтверждения email по ссылке из письма.
Для гладкого входа можно отключить: **Authentication → Providers → Email →**
выключить **«Confirm email»**. Тогда пользователи входят сразу после регистрации.
Если оставить включённым — после регистрации появляется подсказка «подтвердите email».

## Вход по телефону (опционально)
По умолчанию вход по **email**. Для входа по номеру телефона (частый кейс в УЗ)
нужен SMS-провайдер (платно): **Authentication → Providers → Phone**. После
настройки можно расширить логику на `signInWithOtp({ phone })`. Сейчас не включено.

## Что куда сохраняется
| Данные | Таблица | Кто видит |
|---|---|---|
| Профиль (имя, телефон, город, компания) | `profiles` | только сам пользователь (+ владелец/админ) |
| Сохранённые расчёты | `calculations` | только владелец расчёта (+ владелец/админ) |
| Заявки (WhatsApp-форма) | `leads` | вставлять может любой; читать — только владелец/админ |

Подробности схемы и политик — в `supabase/schema.sql`. Аудит исходного состояния —
в `BACKEND_AUDIT.md`.
