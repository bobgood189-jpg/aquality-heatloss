## Что сделано по шагам

**ШАГ 0 — диагностика** → `PERF_BASELINE.md`. Ранних фатальных JS-ошибок не было:
«вечный чёрный экран» объяснялся стоимостью загрузки (1.5 МБ HTML с мегабайтом
инлайн-JS) + JIT-компиляцией Tailwind в браузере до первой отрисовки.

**ШАГ 1 — Tailwind CDN → статика.** `tailwind.config.js` (конфиг перенесён 1:1),
сборка `npm run build:css` → `assets/tailwind.css` (26 КБ вместо ~350 КБ
JIT-компилятора + секунды CPU). Динамических классов-конструкторов
(`bg-${...}`, `classList.add(переменная)`) в коде нет — все классы собираются из
полных литеральных строк, сканер их видит; safelist пуст. Проверено сравнением
ПОЛНЫХ наборов правил CDN-JIT vs статика: **потерь 0**.
⚠️ Важная находка: Play CDN молча НЕ генерировал ~226 групп правил (все
opacity-модификаторы кастомных цветов: `text-amber`, `bg-amber/15`, `bg-w800/40`,
градиенты и часть responsive-вариантов) — известный баг Play CDN с кастомной
палитрой. Статическая сборка генерирует их корректно, поэтому местами сайт стал
выглядеть так, как ЗАДУМАНО разметкой (янтарные подзаголовки «всего здания»,
пилюли бейджей «Доступно в MAX» и т.п.). Это не потеря, а починка.
Каскад воспроизведён точно: CDN вставлял стили в КОНЕЦ head — статический линк
стоит там же (после site.css), проверено поэлементно.

**ШАГ 2 — мгновенный первый экран.** Критический CSS (~2 КБ: фон обеих тем,
кометы) — инлайном; весь JS — defer; в head синхронным остался только крошечный
скрипт темы+tier (обязан выполниться до рендера — анти-FOUC). CSS-заглушка фона:
градиенты hero (уже были в разметке) + 4 «кометы» `.bg-comets` (только
transform/opacity), гаснут по `html.webgl-on`. Шрифты: самостоятельный хостинг
(`assets/fonts/*.woff2`, variable, unicode-range, `font-display:swap`) +
`<link rel="preload">` для двух основных файлов Manrope; Google Fonts CDN удалён.

**ШАГ 3 — безопасный отложенный WebGL.** Шейдерный фон hero стартует после
`load`+`requestIdleCallback` (fallback setTimeout 300 мс) ЛИБО по первому
скроллу/тапу; появляется плавным opacity 0→0.72 за 0.8 с. Отказ WebGL2, падение
компиляции шейдера, `webglcontextlost` → тихий `console.warn` и CSS-фон навсегда.
Three.js (127 КБ) больше вообще не грузится на старте: `ensureThree()` подключает
его при первом реальном открытии 3D (`VIEW3D.open`, `init3DView`). Прежние
оптимизации (pixelRatio cap, FPS throttle, IntersectionObserver) сохранены.

**ШАГ 4 — уровни качества.** PerfManager расширен: `saveData → low`, tier
зеркалится в `document.documentElement.dataset.tier`, PR_CAP medium = 1.
Формула продублирована в раннем инлайн-скрипте head (CSS-гейты `html.tier-*`
работают с первого кадра). **low → WebGL не инициализируется вообще** (кометы
остаются постоянным фоном), mid → pixelRatio 1 / частицы 0.3 / FPS 24-30 (как
было), high → без изменений. С сайта ничего не удалено.

**ШАГ 5 — ленивые модули и медиа.** Supabase уже грузился динамически (проверено:
рендер не блокирует, UI нейтрален до ответа). JS-инжектируемым `<img>` добавлены
`loading="lazy" decoding="async"`. GSAP/Lenis — defer, сцены регистрируются на
DCL (после defer-цепочки ≈ before-load), скролл-эффекты layout в цикле не читают
(PerfManager). Отдельный editor.js не выделялся сознательно: инлайн-onclick по
всему HTML зовут функции редактора глобально, и import() по доскроллу дал бы
гонки; выигрыш уже забрал defer-вынос всего JS.

**ШАГ 6 — разгрузка DOM.** Весь JS (1085 КБ) вынесен из index.html:
`assets/app.js` (PerfManager + приложение + hviz + SX workspace, порядок блоков
сохранён, логика не менялась). CSS (129 КБ) → `assets/site.css`. index.html:
20 389 строк → ~1 000 строк, 1468 КБ → 88 КБ. Секциям ниже первого экрана —
`content-visibility:auto` + `contain-intrinsic-size:auto <высота>` (значения =
реальная высота минус паддинги; скроллбар: 7548px против эталонных 7542px, ±6px).
Секции с канвасами/формами (#calculator, #heat-viz-section) сознательно не
трогались — их JS меряет размеры на init.

**ШАГ 7 — Netlify.** `_headers`: `/assets/*` — `max-age=31536000, immutable`;
`/index.html` и `/` — `must-revalidate`. Ссылки на ассеты версионированы
(`?v=…`): ⚠️ **при изменении файлов в assets поднимайте ?v= в index.html** —
иначе постоянные посетители год не увидят обновления. Preconnect: Supabase,
cdnjs, jsdelivr. Единственные внешние скрипты — GSAP/ScrollTrigger/Lenis (defer,
некритичные: их отказ не роняет UI — каждый init в try/catch).

## Починенные попутно баги продакшена

1. **«Гигантская звезда»**: стили `#aq-nav-sub`/`#floating-sub` инжектировались
   из JS — на слабых устройствах секунды висела огромная нестилизованная звезда
   SVG. Теперь статическое зеркало этих правил в site.css (JS повторная инжекция
   безвредна).
2. **CLS 0.19 → 0**: в `#nav-auth-wrap` добавлен статический скелет кнопок
   «Войти/Регистрация» (та же разметка, что рендерит initAuthNav) — навбар больше
   не прыгает, когда JS досыпает кнопки.
3. Входная анимация hero не переигрывается, если страница видна дольше 2.5 с
   (иначе на медленной сети контент «дёргался» бы при позднем DCL).
